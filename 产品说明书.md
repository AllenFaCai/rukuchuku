## 1. 产品概述

### 1.1 产品简介

库存管理系统 v3.1 是一款基于 Web 的可视化库存管理工具，专为中小型店铺设计。系统采用纯 Python 标准库开发，无需复杂依赖，一键即可启动使用。

### 1.2 核心特性

- **Web 可视化界面**：基于 Bootstrap 5 的现代化 UI，响应式设计，支持桌面和移动端
- **SQLite 数据库**：高性能本地数据库，支持 WAL 模式，确保并发读取效率
- **条码扫描支持**：兼容手持扫码枪，支持实时条码识别
- **批量操作**：支持批量入库/出库，提升操作效率
- **完整的 CRUD**：产品增删改查、库存调整、操作日志全程记录
- **数据导入导出**：支持 Excel 格式，兼容旧版数据
- **零外部依赖**：仅需 Python 标准库 + openpyxl（处理 Excel）

### 1.3 适用场景

- 小型零售店铺库存管理
- 仓库出入库记录
- 商品条码管理与查询
- 库存盘点与调整

## 3. 安装与启动

### 3.1 获取系统文件

系统文件结构如下：

```
pythonProject1/
├── app.py                  # 主程序（Web 服务器 + 业务逻辑）
├── create_templates.py     # 模板生成脚本
├── import_excel.py         # Excel 数据导入工具
├── start.sh                # 一键启动脚本（macOS/Linux）
├── requirements.txt        # 依赖列表
├── stock.db                # SQLite 数据库（首次运行自动生成）
├── templates/              # HTML 模板目录（首次运行自动生成）
│   ├── base.html
│   ├── login.html
│   ├── dashboard.html
│   ├── products.html
│   ├── product_edit.html
│   ├── stock_in.html
│   ├── stock_out.html
│   ├── batch_stock.html
│   ├── stock_manage.html
│   ├── logs.html
│   ├── import.html
│   └── 404.html
├── 产品说明书.md            # 本文档
└── README.md               # 快速说明
```

### 3.2 安装步骤

**步骤 1：安装 Python 依赖**

```bash
pip install openpyxl
```

**步骤 2：启动系统**

方式一：使用启动脚本（推荐）

```bash
# macOS / Linux
bash start.sh

# Windows
python app.py
```

方式二：直接运行

```bash
python3 app.py
```

**步骤 3：访问系统**

打开浏览器，访问：**http://127.0.0.1:5000**

### 3.3 首次启动

首次启动时，系统会自动：

1. 创建 SQLite 数据库文件（`stock.db`）
2. 初始化数据库表结构
3. 创建默认管理员账户（`admin` / `admin123`）
4. 生成 HTML 模板文件（`templates/` 目录）
5. 如果存在 Excel 文件（`Stock_table.xlsx`、`库存表.xlsx`），自动导入数据

## 5. 功能模块详细说明

### 5.1 数据仪表盘

**访问路径**：首页 `/`

数据仪表盘是系统的主页面，提供库存数据的全局概览。

#### 5.1.1 统计卡片

页面顶部显示四个核心统计指标：

| 指标 | 说明 |
|------|------|
| 产品总数 | 系统中注册的产品 SKU 数量 |
| 库存总量 | 所有产品的库存数量总和 |
| 今日操作 | 当天的入库和出库操作次数 |
| 低库存预警 | 库存数量在 1-4 之间的产品数量 |

#### 5.1.2 最近操作记录

显示最近 10 条操作日志，包括：
- 操作时间
- 条形码
- 商品名称
- 变动数量
- 操作类型（入库/出库/调整）
- 操作状态

#### 5.1.3 低库存预警列表

显示库存数量在 1-4 之间的产品，按库存数量升序排列，提醒用户及时补货。

#### 5.1.4 分类与品牌分布

- **产品分类分布**：显示各产品分类的数量统计（最多显示 8 个）
- **品牌分布**：显示各品牌的数量统计（最多显示 8 个）

### 5.2 产品管理

**访问路径**：`/products`

产品管理模块提供产品的完整生命周期管理功能。

#### 5.2.1 产品列表

**功能特性**：
- 分页显示，每页 20 条记录
- 支持多条件搜索（条码、货号、名称模糊匹配）
- 支持按分类筛选
- 支持按品牌筛选
- 显示产品详情：条码、货号、名称、品牌、尺码、分类、库存数量

**操作按钮**：
- **编辑**：跳转到产品编辑页面
- **删除**：删除产品（需二次确认）

#### 5.2.2 新增产品

点击"新增产品"按钮，填写以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 条码/19码 | 是 | 产品的唯一标识码，不可重复 |
| 货号 | 是 | 产品货号/SKU |
| 商品名称 | 是 | 产品名称 |
| 品牌 | 否 | 产品品牌 |
| 尺码 | 否 | 产品尺码（如 S/M/L/XL）|
| 分类 | 否 | 产品分类，默认"未分类" |
| 备注 | 否 | 备注信息 |

**验证规则**：
- 条码、货号、名称为必填项
- 条码不可与已有产品重复
- 提交成功后自动返回产品列表

#### 5.2.3 编辑产品

在产品列表中点击"编辑"按钮，进入产品编辑页面。

可修改所有产品字段，修改后点击"保存"提交。系统会自动更新产品的最后修改时间。

#### 5.2.4 删除产品

在产品列表中点击"删除"按钮，系统会弹出确认对话框。

**警告**：删除产品会同时删除该产品的库存记录和相关操作日志，此操作不可撤销。

### 5.3 入库操作

**访问路径**：`/stock/in`

入库操作模块用于将产品添加到库存中。

#### 5.3.1 操作流程

1. **扫描/输入条码**：在条码输入框中扫描条码或手动输入
2. **系统识别**：
   - 如果条码已存在，自动显示产品信息
   - 如果条码未知，显示手动建档表单
3. **确认入库**：输入入库数量，点击"确认入库"
4. **结果显示**：显示入库结果，包括操作前后库存数量

#### 5.3.2 条码输入方式

- **扫码枪**：直接扫描条码，系统自动识别
- **手动输入**：在输入框中手动输入条码，按回车或点击查询

#### 5.3.3 已知产品入库

当扫描到已存在的条码时：
- 自动显示产品信息（货号、名称、品牌、尺码）
- 显示当前库存数量
- 输入入库数量后点击"确认入库"

#### 5.3.4 未知产品建档

当扫描到未注册的条码时：
- 显示手动建档表单
- 必填字段：条码、货号、名称
- 选填字段：品牌、尺码、入库数量
- 填写完成后点击"建档并入库"
- 系统会同时完成产品建档和入库操作

#### 5.3.5 实时条码查询

系统提供条码查询 API：`/api/barcode/{条码}`

- 返回 JSON 格式的产品信息
- 前端可实时调用，无需刷新页面

---

### 5.4 出库操作

**访问路径**：`/stock/out`

出库操作模块用于从库存中移除产品。

#### 5.4.1 操作流程

1. **扫描/输入条码**：在条码输入框中扫描条码或手动输入
2. **系统识别**：自动显示产品信息和当前库存
3. **输入数量**：填写出库数量
4. **库存检查**：系统自动检查库存是否充足
5. **确认出库**：点击"确认出库"
6. **结果显示**：显示出库结果

#### 5.4.2 库存不足处理

当出库数量大于当前库存时：
- 系统显示"库存不足"错误提示
- 提示当前库存数量
- 阻止出库操作

#### 5.4.3 不存在的条码

当输入的条码不存在时：
- 系统显示"条码不存在"错误提示
- 提示用户先在"入库操作"中建档

---

### 5.5 批量出入库

**访问路径**：`/stock/batch`

批量出入库模块支持一次性处理多个条码，大幅提升操作效率。

#### 5.5.1 操作模式

系统支持两种批量操作模式：

| 模式 | 说明 |
|------|------|
| 批量入库 | 将多个产品批量添加到库存 |
| 批量出库 | 将多个产品批量从库存中移除 |

#### 5.5.2 输入格式

在文本框中输入条码，支持以下格式：

**格式一：每行一个条码**
```
9787111636665
9787111636672
9787111636689
```

**格式二：条码 + 数量（空格分隔）**
```
9787111636665 5
9787111636672 3
9787111636689 10
```

**格式三：混合格式**
```
9787111636665
9787111636672 3
9787111636689 10
```

#### 5.5.3 默认数量

如果条码后没有指定数量，系统使用"默认数量"字段的值（默认为 1）。

#### 5.5.4 执行结果

批量操作完成后，系统显示详细的结果报告：

- **操作摘要**：成功数量、失败数量、总计数量
- **详细列表**：每个条码的处理结果
  - 成功：显示产品信息、操作数量、操作前后库存
  - 失败：显示失败原因（条码不存在、库存不足等）

#### 5.5.5 失败处理

批量操作中，单个条码的失败不会影响其他条码的处理。系统会：
- 跳过失败的条码
- 继续处理剩余条码
- 在结果中详细记录每个失败原因

### 5.6 库存管理

**访问路径**：`/stock/manage`

库存管理模块提供库存的查看和调整功能。

#### 5.6.1 库存总览

- 分页显示所有产品的库存信息
- 每页 20 条记录
- 支持按条码、货号、名称搜索

#### 5.6.2 库存状态标记

系统根据库存数量自动标记状态：

| 状态 | 条件 | 说明 |
|------|------|------|
| 正常 | 库存 ≥ 5 | 库存充足 |
| 低库存 | 1 ≤ 库存 < 5 | 需要补货 |
| 缺货 | 库存 = 0 | 无库存 |

#### 5.6.3 手动库存调整

点击产品行中的"调整"按钮，可以手动修改库存数量。

**使用场景**：
- 库存盘点后修正
- 损耗/报废处理
- 其他特殊情况

**操作步骤**：
1. 点击"调整"按钮
2. 在弹出的对话框中输入新的库存数量
3. 填写调整原因（必填）
4. 点击"确认调整"

**注意事项**：
- 库存数量不能为负数
- 系统会自动记录调整日志
- 调整原因会记录在操作日志中

### 5.8 数据导入

**访问路径**：`/import`

数据导入模块支持从 Excel 文件导入产品数据。

#### 5.8.1 支持的文件格式

- `.xlsx`（Excel 2007 及以上）
- `.xls`（Excel 97-2003）

#### 5.8.2 导入规则

**表头识别**：
系统自动识别以下表头字段：
- 条码/19码、条形码、ISBN码
- 货号
- 商品名称
- 品牌
- 尺码
- 库存数量
- 备注

**数据处理**：
- 如果条码已存在，更新产品信息（仅更新非空字段）
- 如果条码不存在，创建新产品

**多 Sheet 支持**：
- 系统会遍历 Excel 文件中的所有 Sheet
- 每个 Sheet 独立处理

#### 5.8.3 操作步骤

1. 点击"选择文件"按钮，选择 Excel 文件
2. 也可以将文件拖拽到上传区域
3. 点击"开始导入"按钮
4. 等待导入完成
5. 查看导入结果（新增数量、更新数量）

#### 5.8.4 注意事项

- 导入过程中请勿关闭页面
- 大文件导入可能需要较长时间
- 导入失败会显示错误信息

## 6. 数据库结构

### 6.1 数据库概述

- **数据库引擎**：SQLite
- **存储模式**：WAL（Write-Ahead Logging）模式
- **文件位置**：项目目录下的 `stock.db`
- **字符编码**：UTF-8

### 6.2 表结构

#### 6.2.1 products 表（产品信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| barcode | TEXT | 条码，唯一，不可为空 |
| sku | TEXT | 货号，不可为空 |
| name | TEXT | 商品名称，不可为空 |
| brand | TEXT | 品牌，默认空 |
| size | TEXT | 尺码，默认空 |
| category | TEXT | 分类，默认"未分类" |
| remark | TEXT | 备注，默认空 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

**索引**：
- `idx_products_barcode`：barcode 字段
- `idx_products_sku`：sku 字段
- `idx_products_category`：category 字段
- `idx_products_brand`：brand 字段

#### 6.2.2 stock 表（库存信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| product_id | INTEGER | 产品 ID，唯一，外键关联 products 表 |
| quantity | INTEGER | 库存数量，默认 0 |
| updated_at | TIMESTAMP | 更新时间 |

**约束**：
- `product_id` 与 `products.id` 建立外键关系
- 删除产品时，自动删除对应的库存记录（CASCADE）

#### 6.2.3 logs 表（操作日志）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| barcode | TEXT | 条码，不可为空 |

## 7. 技术架构

### 7.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端 | Python 标准库 | `http.server` + `sqlite3` |
| 前端 | Bootstrap 5 | 响应式 UI 框架 |
| 图标 | Font Awesome 6 | 图标库 |
| 数据库 | SQLite | 本地文件数据库 |
| 模板 | 自定义引擎 | Jinja2 风格语法 |

### 7.2 架构特点

**纯标准库实现**：
- 使用 `http.server.HTTPServer` 作为 Web 服务器
- 使用 `BaseHTTPRequestHandler` 处理 HTTP 请求
- 使用 `sqlite3` 进行数据库操作
- 无需安装 Flask、Django 等 Web 框架

**自定义模板引擎**：
- 支持 `{% extends %}` / `{% block %}` 模板继承
- 支持 `{% for %}` / `{% endfor %}` 循环
- 支持 `{% if %}` / `{% elif %}` / `{% else %}` / `{% endif %}` 条件判断
- 支持 `{{ variable }}` 变量输出
- 支持 `| safe`、`| truncate` 过滤器

**会话管理**：
- 基于内存的会话存储
- 使用 HTTP Cookie（`sid`）标识会话
- 会话数据存储在服务器内存中

**数据库优化**：
- WAL 模式提升并发读取性能
- 合理的索引设计
- 外键约束保证数据完整性

### 7.3 安全机制

- 密码 SHA-256 加密存储
- 会话验证防止未授权访问
- SQL 参数化查询防止注入攻击
- HTML 转义防止 XSS 攻击

## 8. 常见问题解答

### 8.1 启动问题

**Q：启动时报错"Address already in use"**

A：端口 5000 已被占用。解决方案：
1. 关闭占用端口的程序
2. 或修改 `app.py` 中的 `PORT` 变量为其他端口

**Q：启动后无法访问 http://127.0.0.1:5000**

A：可能原因：
1. 防火墙阻止了访问
2. 浏览器缓存问题，尝试清除缓存或使用无痕模式
3. 检查终端是否有错误信息

**Q：首次启动没有自动生成数据库**

A：检查是否有写入权限。系统会尝试在项目目录创建 `stock.db`，如果失败会使用 `/tmp/stock.db`。

### 8.2 登录问题

**Q：忘记密码怎么办**

A：默认账户密码为 `admin123`。如果修改过密码，可以删除 `stock.db` 文件，重新启动系统会创建新的默认账户。

**Q：登录后立即退出**

A：可能是浏览器禁用了 Cookie。请启用 Cookie 后重试。

### 8.3 数据问题

**Q：导入 Excel 失败**

A：可能原因：
1. 文件格式不正确（仅支持 .xlsx 和 .xls）
2. 文件损坏
3. 文件正在被其他程序打开
4. 数据格式异常

**Q：条码查询不到产品**

A：可能原因：
1. 产品未在系统中建档
2. 条码输入错误
3. 使用"入库操作"中的手动建档功能添加产品

**Q：库存数量不正确**

A：可以使用"库存管理"中的"调整"功能手动修正库存，并填写调整原因。

### 8.4 性能问题

**Q：系统运行缓慢**

A：可能原因：
1. 数据库文件过大（超过 1GB）
2. 浏览器缓存问题
3. 系统资源不足

解决方案：
1. 定期清理旧的操作日志
2. 清除浏览器缓存
3. 关闭其他占用资源的程序

**Q：大量数据导入很慢**

A：SQLite 在大量数据写入时性能会下降。建议：
1. 分批导入（每次不超过 10000 条）
2. 在系统空闲时进行导入
3. 导入完成后重启系统
